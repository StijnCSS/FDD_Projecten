We zijn er bijna

Tijdens de minor heb ik ontzettend veel geleerd. Ik heb aan serieuze projecten gewerkt waarin ik echt de diepte in ben gegaan met CSS, JavaScript en API’s. Vooral het creatieve coderen vind ik leuk. Ik merk dat ik graag met CSS werk en daar vette dingen mee kan maken. Wel heb ik soms nog moeite met complexere JavaScript-logica, maar ik wil dit juist gaan toepassen tijdens de Meesterproef om meer interactieve elementen toe te voegen.

We hebben ook veel aandacht besteed aan progressive enhancement. Daar heb ik superveel aan gehad, omdat ik heb gezien hoe krachtig HTML, CSS en JavaScript kunnen zijn als je het goed inzet. Het was fijn om talks en lessen te krijgen van experts die alles helder en met enthousiasme konden uitleggen.

Ik heb het vak API gehaald, al vind ik de stof soms nog lastig. Tijdens de Meesterproef ga ik me daar waarschijnlijk minder op richten, omdat we samenwerken in een team. Ik wil me liever focussen op interactieve en geanimeerde onderdelen van het project.

Ik ben trots op alle projecten waar ik aan heb gewerkt. Ik probeer altijd mijn eigen flair toe te voegen. Ik ben flexibel in mijn conceptontwikkeling als iets niet werkt, begin ik gewoon opnieuw en zoek ik naar een betere oplossing.

met design ben ik goed in de tools en dat zorgt ervoor dat ik vaak kan maken wat ik wil omdat ik precies weet wat ik kan doen met Photoshop, After Effects etc. Met code mis ik soms nog de tool skills. Ik wil meer tutorials gaan kijken en echt weer leren zoals ik vroeger deed.

Leerdoel 1 MOTION

ik wil meer doen met motion. Ik hoop tijdens de meesterproef deze taak op mij te nemen. Tijdens de WN van Cassie werd ik erg geinspireerd en ik wil kijken of ik wat met GSAP of soorgelijke tools kan maken voor de meesterproef.

Leerdoel 2 PLANNEN

ik doe vaak dingen last minute vooral als ik het wat minder leuk vind. Ik wil tijdens de meesterpreof weekelijks/dagelijks mijn documentatie bijhouden zodat ik geen nacht hoef door te halen om alles te halen.

Leerdoel 3 SAMENWERKEN

Samenwerken gaat mij meestal goed af, zeker omdat er in de klas veel verschillende mensen zitten met verschillende skills. Juist daarom wil ik aan het begin van het project extra aandacht besteden aan een groep maken die elkaar versterkt en daar een goede taakverdeling mee maken, zodat iedereen zich kan focussen op zijn of haar eigen onderdeel van het project. Tijdens de hack-a-thon merkte ik dat samenwerken via GitHub nog best rommelig was. We werkten tegelijk in dezelfde bestanden en kregen daardoor veel merge-conflicten. Ik wil beter leren samenwerken met GitHub en begrijpen hoe we als team structuur kunnen aanbrengen in onze workflow. Ook wil ik vaker korte check-ins houden, bijvoorbeeld een paar keer per dag om te zorgen dat we op één lijn blijven en elkaar kunnen helpen waar nodig.
